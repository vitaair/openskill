# 开发感知编译器 Playbook

## 目录

- 任务编译器
- Bug 压缩器
- 实现边界卡
- 验证闭环
- Review 输出
- Agent 交接

## 任务编译器

用于把用户需求改写成可执行开发任务。

```markdown
任务名：
……

目标：
用户最终要达成……

背景：
当前系统/页面/模块……

工程入口：
- 文件/目录/命令……

实现边界：
- 可以改……
- 不要改……

质量标准：
- 功能……
- 交互/性能/安全/兼容……

验收：
- 命令……
- 截图/浏览器步骤……

缺失信息：
- 只列会影响实现方案的问题
```

## Bug 压缩器

用于把报错、日志和用户描述转成定位信息。

```markdown
现象：
……

复现：
1. ……
2. ……

最早失败点：
- 命令/页面操作……
- 错误……
- 文件/行号……

期望 vs 实际：
- 期望……
- 实际……

怀疑路径：
- 数据流……
- 状态流……
- 接口契约……
- 环境/权限/时序……

下一步：
- 读……
- 运行……
```

## 实现边界卡

用于防止过度修改。

```markdown
写入范围：
- ……

读取范围：
- ……

不动：
- 公共 API……
- 设计 token……
- 数据库迁移……
- 用户已有改动……

最小改动策略：
- 优先复用……
- 避免新依赖……
- 避免无关重构……
```

## 验证闭环

```markdown
验证矩阵：
| 风险 | 验证方式 | 结果 |
|---|---|---|
| 功能行为 | 测试/复现命令 | …… |
| 类型/构建 | typecheck/build | …… |
| UI 回归 | 浏览器/截图 | …… |
| 边界条件 | 单测/手动步骤 | …… |

无法验证：
- 原因……
- 剩余风险……
```

## Review 输出

```markdown
发现：
- [P1] 文件:行号 - 问题标题
  触发条件：……
  影响：……
  建议：……

测试缺口：
- ……

未发现问题：
- 如果没有发现问题，明确说明，并列出审查范围。
```

## Agent 交接

```markdown
给接手 Agent：

任务：
……

当前状态：
- 已读……
- 已改……
- 已验证……

证据：
- 文件……
- 命令输出摘要……
- 截图/日志……

写入边界：
- 你负责……
- 不要碰……

下一步：
1. ……
2. ……

完成回报：
- 文件清单
- 验证结果
- 剩余风险
```
