# 架构图模板库

来源参考：https://www.cnblogs.com/wuhuixuri/articles/18342003

说明：本目录将文章中的八类架构图抽象为软著材料可复用模板。模板只复用图类型、层级和布局方法，不复制文章图片内容。

| 模板 | 类型 | 用途 |
|---|---|---|
| `ARCH01_业务架构图模板` | 业务架构 | 用于表达业务目标、业务能力、核心流程、组织角色和指标闭环。 |
| `ARCH02_产品架构图模板` | 产品架构 | 用于表达产品端、功能域、平台能力、运营能力和路线规划。 |
| `ARCH03_系统架构图模板` | 系统架构 | 用于表达用户、接入层、服务层、数据层、外部系统和运维支撑。 |
| `ARCH04_数据架构图模板` | 数据架构 | 用于表达数据源、采集、处理、治理、存储、服务和应用消费。 |
| `ARCH05_技术架构图模板` | 技术架构 | 用于表达前端技术、服务技术、中间件、数据组件、基础设施和运维安全。 |
| `ARCH06_应用架构图模板` | 应用架构 | 用于表达前端 UI、展示层、应用服务层、基础服务层和基础设施层。 |
| `ARCH07_功能架构图模板` | 功能架构 | 用于表达系统功能域、一级功能、二级功能和横向公共能力。 |
| `ARCH08_信息架构图模板` | 信息架构 | 用于表达首页、导航、栏目、详情页、操作页和辅助信息之间的组织关系。 |
| `ARCH09_大数据平台总体架构复刻模板` | 平台总体架构 | 基于用户手动调整后的 PJ1 架构图沉淀，适用于大数据平台、数据治理、主数据管理、数据资产类项目。 |

## 使用规则

- 生成正式材料前，必须用项目事实替换模板占位文本。
- 软著提交图必须保留 `.drawio` 源文件，并导出 SVG/PNG。
- 平台型项目优先使用数据架构图、系统架构图、技术架构图和应用架构图模板。
- 流程类图仍按 UML 活动图规则生成，不用架构图模板代替。
