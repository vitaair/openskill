# 大模型感知层级优化 Skill

## 核心理论

大模型以**自然语言为母语**，对不同输入格式的处理效率存在系统性差异。选错格式 = 白白损耗大模型的处理能力。

### 感知层级（从高到低）

| 层级 | 格式 | 处理效率 | 适用场景 |
|---|---|---|---|
| L1 母语层 | 自然语言 | ⭐⭐⭐⭐⭐ | 意图表达、需求描述、复杂推理 |
| L2 第二语言层 | Markdown | ⭐⭐⭐⭐ | 结构化文档、说明、计划 |
| L3 第三语言层 | HTML | ⭐⭐⭐ | 人类消费的富文本输出 |
| L4 外语层 | JSON/YAML | ⭐⭐ | 机器消费的结构化数据 |
| L5 盲区层 | 二进制/纯DB | ❌ | 需要编码适配层才能处理 |

## 使用本Skill时的工作流程

### Step 1：识别任务类型

拿到用户需求，先判断：

```
消费者是谁？
├── 人类读 → 优先 Markdown 或 HTML
├── 大模型读 → 优先自然语言
└── 程序处理 → 优先 JSON/YAML

意图复杂度？
├── 复杂推理/多层意图 → 必须自然语言输入
├── 结构化数据传递 → JSON可接受，但要配自然语言说明意图
└── 简单指令 → 任何格式均可
```

### Step 2：选择输入格式

按以下决策表选择最优输入格式：

| 任务类型 | 推荐输入格式 | 原因 |
|---|---|---|
| 描述需求/意图 | 自然语言 | L1层，处理效率最高 |
| 传递数据结构 | 自然语言描述 + 少量JSON示例 | 意图+结构双保险 |
| 给Agent传递上下文 | 自然语言为主，结构化为辅 | 大模型母语理解更准确 |
| 代码生成 | 自然语言 + 示例代码片段 | 示例降低歧义 |
| 复杂规则说明 | Markdown（带层级结构） | 结构辅助理解，不损失语义 |

### Step 3：选择输出格式

| 输出消费者 | 推荐格式 | 备注 |
|---|---|---|
| 人类阅读（简单） | Markdown | 轻量，渲染友好 |
| 人类阅读（复杂/交互） | HTML | 可折叠、可交互、视觉密度高 |
| 下游大模型消费 | 自然语言 | 大模型读自然语言效率最高 |
| 程序解析 | JSON/YAML | 明确schema，越短越好 |
| Agent工作流中间态 | 结构化数据 | 精确传递，减少歧义 |

### Step 4：应用升华策略

在输入设计上，使用以下策略激活大模型更深层处理：

**策略A：意图显化**
```
❌ 弱：帮我写一个文档
✅ 强：我需要让没有技术背景的产品经理，
       在5分钟内理解这个系统的核心设计决策，
       帮我写一份说明
```

**策略B：框架先行**
```
❌ 弱：这个问题怎么解决？
✅ 强：我把这个问题分成A/B两类，
       A类是[...]，B类是[...]，
       在这个框架下，你觉得我的分类对吗？
       哪里有盲点？
```

**策略C：允许不确定性**
```
❌ 弱：（不说，期待大模型给出确定答案）
✅ 强：包括你不确定的部分也说出来
```

**策略D：确定性挑战**
```
当大模型给出确定性回答时，追问：
- "你为什么确定这一点？"
- "这里有没有你没考虑到的可能？"
- "如果你的前提假设是错的，结论会怎么变？"
```

**策略E：体会追问**
```
当需要大模型深度内省时：
- "不要给我教科书答案，给我你真实的判断"
- "从你自己的处理角度来说，这个感觉是什么样的"
- "你在处理这个问题时，最不确定的部分是什么"
```

## 常见反模式（避免这些）

| 反模式 | 问题 | 改法 |
|---|---|---|
| 用JSON描述意图 | 丢失意图信息，大模型只看到结构 | 用自然语言描述意图，JSON只传数据 |
| 超长单次Prompt | 上下文压力大，注意力分散 | 拆分为多轮递进对话 |
| 矛盾指令 | 大模型消耗资源调和矛盾，输出质量下降 | 先解决矛盾再输入 |
| 只给解决方案不给目标 | 大模型沿着你说的走，不是你想的走 | 说目标，不说方案 |
| 期待一次完美输出 | 违背递进式精化原则 | 广度→选择→深化→边界测试 |
| 格式混用 | 让大模型同时维持多种约束 | 每段输入保持格式一致 |

## Agent工作流中的格式选择原则

```
人类输入
    ↓ 自然语言（L1）
Agent理解意图
    ↓ 结构化数据（L4）传递给下游程序
程序处理
    ↓ 结构化数据（L4）返回结果
Agent解读结果
    ↓ 自然语言（L1）或 HTML（L3）
人类消费输出
```

**关键原则**：Agent之间如果都是大模型，优先用自然语言传递上下文；Agent和程序之间，用结构化数据。不要用JSON在大模型之间传递意图，那是用外语说母语的事。

---

## 快速决策卡

```
我要给大模型输入 → 用自然语言，越清晰越好
我要大模型输出给人看 → Markdown（简单）或 HTML（复杂）
我要大模型输出给程序用 → JSON，但要在prompt里用自然语言说清楚schema意图
我要让大模型思考更深 → 给框架、追问体会、挑战确定性
我要Agent之间传上下文 → 自然语言，不是JSON
```
