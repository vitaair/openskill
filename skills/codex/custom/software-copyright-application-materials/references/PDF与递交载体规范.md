# PDF 与递交载体规范

## 基本原则

Markdown、Word 或其他源稿只是编辑载体。对外递交通常以登记系统要求的 PDF、扫描件或在线表单为准。

递交前必须确认：

- PDF 与对应源稿内容一致。
- PDF 可正常打开，中文、图片、表格无缺失或错位。
- 文件名与提交清单一致。
- 在线填报字段与申请表 PDF 一致。
- 签章扫描件为真实文件，不是占位说明。

## 建议 PDF 清单

| 文件 | 说明 |
| --- | --- |
| `01_软件说明书.pdf` 或 `01_系统说明书.pdf` | 由说明书源稿导出，建议含软件全称、版本号和连续页码。 |
| `02_源程序代码.pdf` | 源程序鉴别材料，建议含软件全称、版本号、页码和每页不少于 50 行。 |
| `03_界面截图说明.pdf` | 与截图文件一致，图号和图注清晰。 |
| `04_计算机软件著作权登记申请表.pdf` | 优先使用登记系统导出件。 |
| `05_申请人主体证明.pdf` | 可作说明归档，不能替代真实身份证明扫描件。 |
| `05_附件_营业执照副本复印件_加盖公章.pdf` | 企业申请通常必备，须人工替换为真实盖章扫描件。 |
| `06_代理委托书（委托代理机构时提交）.pdf` | 仅委托代理时提交；自理登记不要上传占位件。 |

## 页眉页脚

- 源程序材料建议页眉体现软件全称、版本号和“源程序代码”或“源程序鉴别材料”。
- 说明书建议页眉或页脚体现软件全称、版本号和连续页码。
- 总页数、每页行数、A4 版式以登记系统和受理窗口当时要求为准。

## 人工检查

自动脚本只能生成或辅助检查 PDF，不能替代人工终检。提交前应人工打开 PDF，重点查看：

- 首页软件名称和版本。
- 页码是否连续。
- 截图是否清晰且未丢失。
- 表格是否跨页错乱。
- 是否仍有“待补”“请按实际修改”“占位”等编辑痕迹。
