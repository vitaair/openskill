# 版本与原创口径

## 默认口径

新申报项目若为首次登记、未基于已登记软件升级，通常使用：

- 版本号：`V1.0.0` 或项目既定版本号。
- 作品说明：原创。

同一项目内必须统一版本写法，不要混用 `V1.0`、`V1.0.0`、`1.0.0`。

## 是否需要升级版本说明

在 `V1.0.0` 且作品说明为“原创”的口径下，通常不单独提交：

- 升级版本说明。
- 前期开发说明。

例外情况：

- 登记系统某字段强制上传。
- 审查补正或代理机构明确要求。
- 项目确实基于既有登记软件升级。

出现例外时，应按真实情况单独撰写，并确保申请表、说明书、源程序页眉和补充说明口径一致。

## 写作注意

- 不要为了“显得完整”主动编造升级历史。
- 不要把父平台名称当作当前软件名称。
- 若软件来自大平台拆分子系统，应在内部说明中记录拆分依据；正式材料聚焦本次申请的软件边界。
