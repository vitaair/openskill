# 软著提交材料清单

## 必备材料

1. 计算机软件著作权登记申请表
   - 以登记系统在线填报并导出的终稿为准。
   - 软件全称、简称、版本号、著作权人、开发完成日期、发表状态等字段须与说明书和源程序材料一致。
2. 软件说明书或系统说明书
   - 覆盖软件用途、运行环境、总体结构、功能模块、操作流程、界面截图、数据对象、权限和运行维护。
   - 图号、菜单名、功能名应与截图和正文一致。
3. 源程序代码鉴别材料
   - 常见口径为前连续 30 页和后连续 30 页；不足 60 页时提交全部。
   - 每页不少于 50 行，正文应为可复制程序文本，不混入需求说明、目录解释或占位说明。
4. 申请人身份证明
   - 企业：营业执照副本复印件并加盖公章。
   - 个人：身份证复印件并签字。
5. 权利归属材料
   - 按实际情况准备委托开发协议、合作开发协议、职务开发证明等。
   - 若无权属特殊情况，不要虚构附件。

## 建议同步准备

- 版本信息页：软件名称、简称、版本号、完成日期、首次发表日期。
- 功能清单：模块列表、入口、主要动作、输入输出、对应源码。
- 截图清单：截图文件名、对应功能、说明书章节或页码。
- 材料核对表：提交前逐项勾选 PDF、签章、命名和一致性。

## 一致性要求

- 软件全称一致：申请表、说明书、源程序页眉、文件名中的软件名称不得混用简称或旧名。
- 版本一致：全套材料统一使用同一种版本写法，例如 `V1.0.0`。
- 时间一致：开发完成日期、首次发表日期、申请日期须与事实和线上填报一致。
- 功能一致：说明书功能、截图、源码模块、申请表功能描述应能互相对应。
- 风险内容清除：不得出现“待补”“请按实际修改”、测试账号、密钥、生产库连接串、真实个人隐私等。
