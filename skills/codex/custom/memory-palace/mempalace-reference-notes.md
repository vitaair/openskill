### M-20260512-002：原文抽屉和索引层分离

- 状态：active
- 区域：档案室
- 标签：#架构 #原文 #索引 #MemPalace
- 触发场景：设计记忆写入、检索、压缩和归档规则时
- 记忆内容：MemPalace 强调 drawer 保存原文，closet 只保存可搜索的紧凑指针。索引用来定位原文，不替代原文。
- 使用方式：本 SKILL 应增加“卡片是索引，不是原文替代”的规则；重要对话、决策和文档应保留来源引用或原文路径。
- 来源：`README.md`、`CLAUDE.md`、`docs/CLOSETS.md`
- 更新时间：2026-05-12

### M-20260512-003：L0-L3 分层唤醒可迁移到 Markdown 记忆

- 状态：active
- 区域：工坊
- 标签：#流程 #分层唤醒 #Agent协作
- 触发场景：Agent 接手项目、压缩上下文、或需要快速加载记忆时
- 记忆内容：MemPalace 使用 L0 身份、L1 核心故事、L2 按主题加载、L3 深度搜索的四层记忆栈，降低启动成本。
- 使用方式：本 SKILL 可把“入口/门厅”作为 L0，“出口/工作台”作为 L1，“地图/展厅/工坊/警示墙”作为 L2，“原始文件/参考仓库/历史归档”作为 L3。
- 来源：`mempalace/layers.py`
- 更新时间：2026-05-12

### M-20260512-004：增量、本地、可修复是长期记忆底线

- 状态：active
- 区域：门厅
- 标签：#规则 #本地优先 #增量 #治理
- 触发场景：设计自动写入、迁移、同步和外部依赖规则时
- 记忆内容：MemPalace 的设计原则包括本地优先、增量写入、隐私边界、后台保存、性能预算和修复机制。长期记忆系统不能依赖不可控外部服务，也不能靠破坏性重建维持一致性。
- 使用方式：本 SKILL 应明确：默认本地文件优先、引用原文、避免破坏性覆盖、自动改动要可审计。
- 来源：`CLAUDE.md`、`mempalace/palace.py`、`mempalace/repair.py`
- 更新时间：2026-05-12

## 2. 可吸收设计

- Wing/Room/Drawer 可映射为：项目/九区/原文或记忆卡片。
- Closet 可映射为：索引卡片、摘要指针、标签和触发场景。
- L0-L3 可映射为：启动必读、当前交接、按需区域、深度原文。
- Knowledge graph 可映射为：实体关系、决策关系、替代关系和冲突登记。
- Hooks 可映射为：任务结束、上下文压缩前、项目阶段切换时的自动审计。

## 3. 不直接照搬内容

- 不引入 ChromaDB、MCP server 或向量搜索作为本 SKILL 的必需依赖。
- 不把 MemPalace 的 benchmark 声明写成本 SKILL 的能力。
- 不复制上游代码到核心规范，也不在本 SKILL 内嵌上游仓库副本。
- 不把“永远原文保存”绝对化到所有场景；本 SKILL 仍以项目协作、规则、交接和可迁移 Markdown 为主。
