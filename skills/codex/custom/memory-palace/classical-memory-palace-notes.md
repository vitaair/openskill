# 古典记忆宫殿原典吸收笔记

参考文本：`Rhetorica ad Herennium`, Book III

## 1. 核心发现

### M-20260512-001：人工记忆由地点和图像构成

- 状态：active
- 区域：展厅
- 标签：#原典 #记忆宫殿 #loci #images
- 触发场景：解释记忆宫殿的底层结构，或设计 AI/Agent 记忆分类方式时
- 记忆内容：古典记忆术把人工记忆建立在两个部分上：地点和图像。地点提供稳定秩序，图像承载要记的信息。
- 使用方式：AI 记忆宫殿中，九区结构相当于地点，记忆卡片相当于图像；必须先有稳定位置，再放入内容。
- 来源：`Rhetorica ad Herennium`, Book III, memory chapters
- 更新时间：2026-05-12

### M-20260512-002：地点必须有顺序，才能支持正向和反向回忆

- 状态：active
- 区域：工坊
- 标签：#原典 #顺序 #检索
- 触发场景：设计目录结构、索引顺序、启动读取顺序和交接流程时
- 记忆内容：古典方法强调地点要按秩序排列；排列好后，可以从头、从尾或从中间开始回忆。
- 使用方式：AI 记忆宫殿的九区、L0-L3 唤醒、项目索引和交接摘要必须保持稳定顺序；不能把记忆写成无序堆积。
- 来源：`Rhetorica ad Herennium`, Book III
- 更新时间：2026-05-12

### M-20260512-003：图像要醒目、具体、反常，才能唤起记忆

- 状态：active
- 区域：工坊
- 标签：#原典 #图像化 #触发场景
- 触发场景：把抽象规则、风险、决策或流程转成可检索卡片时
- 记忆内容：古典方法认为普通、模糊、常见的图像不容易刺激记忆；异常、清晰、有动作、有差异的图像更容易被记住。
- 使用方式：AI 记忆卡片不能只写泛泛摘要，应写明确标题、触发场景、风险条件、行动方式和来源，让后续 Agent 一眼知道何时使用。
- 来源：`Rhetorica ad Herennium`, Book III
- 更新时间：2026-05-12

### M-20260512-004：记事和记词是两种不同难度

- 状态：active
- 区域：档案室
- 标签：#原典 #抽象 #原文层 #索引层
- 触发场景：判断应保存原文、摘要、索引还是结构化卡片时
- 记忆内容：古典记忆术区分记住事情和逐词记住。记事情更适合图像与地点，逐词记忆更困难，需要更严格训练。
- 使用方式：本 SKILL 应继续区分原文层和索引层。规则、决策、流程可用卡片索引；需要逐字准确的信息必须回到原文层。
- 来源：`Rhetorica ad Herennium`, Book III
- 更新时间：2026-05-12

### M-20260512-005：反复巡行是巩固记忆的必要动作

- 状态：active
- 区域：工坊
- 标签：#原典 #复习 #审计
- 触发场景：设计审计、复盘、交接、定期巡检和项目接续时
- 记忆内容：古典方法要求反复在心中巡行地点以刷新图像。记忆宫殿不是写入一次就结束，而是需要周期性回访。
- 使用方式：AI 记忆宫殿中的轻量审计、阶段审计、迁移审计，就是 Agent 版本的“巡行宫殿”。审计不是额外流程，而是记忆保持机制。
- 来源：`Rhetorica ad Herennium`, Book III
- 更新时间：2026-05-12

## 2. 对本 SKILL 的落地映射

- 地点：九区结构、文件目录、项目索引、L0-L3。
- 图像：记忆卡片、标签、触发场景、风险条件、决策摘要。
- 顺序：读取顺序、写入流程、项目初始化流程、交接流程。
- 醒目性：标题要具体，触发场景要明确，风险要写触发条件。
- 原文：逐词准确内容、合同、口径、代码、命令输出、用户明确原话。
- 巡行：审计、复盘、下次启动提示、阶段交接。
